üí° Key Concepts You Need to Understand

OLED vs. LCD Screens

OLED pixels produce their own light. A black pixel is literally turned off (saving power). LCD screens have a constant backlight behind all pixels ‚Äî black pixels still block light but the backlight stays on. This is why dark mode saves more battery on OLED.

True Black vs. Dark Gray

Google's Material Design recommends dark gray (#121212) over true black (#000000) for dark mode. Why? Pure black next to white text creates very high contrast that can cause halation (a bright glow around text that strains your eyes). Dark gray is easier to read.

Contrast Ratio

The difference in brightness between text and background. WCAG accessibility guidelines recommend at least 4.5:1 for normal text and 3:1 for large text. Too little contrast = hard to read. Too much = eye strain. Dark mode must balance this carefully.

Positive vs. Negative Polarity

Positive polarity = dark text on light background (light mode). Negative polarity = light text on dark background (dark mode). Research shows positive polarity is generally better for readability, but negative polarity can reduce eye strain in low light.

Android Theme System

Android uses themes.xml (or styles.xml) to define app-wide colors. You create a "night" variant in res/values-night/ to define dark mode colors. Android switches automatically based on system setting.

DayNight Theme

Android's Theme.MaterialComponents.DayNight is the base theme that enables automatic day/night switching. Your app inherits from this and overrides specific colors for each mode.

Phase 1: Implement Dark Mode in a Sample App

1

Create a New Android Project

Create a new project with Empty Views Activity. Name it DarkModeStudy. Choose API 21+ (Android 5.0) as minimum SDK.

2

Set Up the DayNight Theme

Open res/values/themes.xml and ensure your base theme extends DayNight:

XML 
<!-- res/values/themes.xml (Light Mode) -->
<resources>
  <style name="Theme.DarkModeStudy"
         parent="Theme.MaterialComponents.DayNight.DarkActionBar">
    <item name="colorPrimary">#1976D2</item>
    <item name="colorOnPrimary">#FFFFFF</item>
    <item name="android:colorBackground">#FFFFFF</item>
    <item name="colorOnBackground">#1C1B1F</item>
    <item name="colorSurface">#FFFFFF</item>
    <item name="colorOnSurface">#1C1B1F</item>
  </style>
</resources>
3

Create the Night Theme Variant

Right-click on res/values ‚Üí New ‚Üí Values Resource Directory. Name it values-night. Create themes.xml inside it:

XML 
<!-- res/values-night/themes.xml (Dark Mode) -->
<resources>
  <style name="Theme.DarkModeStudy"
         parent="Theme.MaterialComponents.DayNight.DarkActionBar">
    <item name="colorPrimary">#90CAF9</item>
    <item name="colorOnPrimary">#003258</item>
    <item name="android:colorBackground">#121212</item>
    <item name="colorOnBackground">#E3E3E3</item>
    <item name="colorSurface">#1E1E1E</item>
    <item name="colorOnSurface">#E3E3E3</item>
  </style>
</resources>
üëÄ
Notice: We used #121212 (dark gray) instead of #000000 (true black) following Material Design guidelines. You can experiment with both!
4

Add a Toggle Button in Your App

Let users switch between dark and light mode with a button. Add this to your Activity:

Kotlin 
// In your Activity
import androidx.appcompat.app.AppCompatDelegate

// Toggle dark mode on button click:
toggleButton.setOnClickListener {
    val currentMode = AppCompatDelegate.getDefaultNightMode()
    if (currentMode == AppCompatDelegate.MODE_NIGHT_YES) {
        AppCompatDelegate.setDefaultNightMode(
            AppCompatDelegate.MODE_NIGHT_NO
        )
    } else {
        AppCompatDelegate.setDefaultNightMode(
            AppCompatDelegate.MODE_NIGHT_YES
        )
    }
}

Phase 2: Battery Measurement

5

Simple Battery Level Monitoring

The simplest way to observe battery impact. On an Android device or emulator:

  1. Go to Settings ‚Üí Battery and note the current battery percentage
  2. Open your app in Light Mode at full brightness
  3. Leave it on-screen for 15-20 minutes (don't interact with it)
  4. Note the battery percentage again
  5. Charge back to the same level, then repeat with Dark Mode
  6. Compare the battery drain
‚ö†Ô∏è
Important: This works best on a physical device with an OLED screen. On an emulator or LCD device, the difference will be much smaller. If you only have an emulator, that's okay ‚Äî note it in your methodology and focus on the theoretical explanation instead.
6

Using Battery Stats via ADB (Advanced)

For more precise data, use Android Debug Bridge (ADB) commands. Open the Terminal in Android Studio (bottom panel, "Terminal" tab):

Terminal 
# Reset battery stats
adb shell dumpsys batterystats --reset

# Use your app in Light mode for 15 min, then:
adb shell dumpsys batterystats > light_mode_stats.txt

# Reset and repeat for Dark mode:
adb shell dumpsys batterystats --reset
# (use app in Dark mode for 15 min)
adb shell dumpsys batterystats > dark_mode_stats.txt

Search these files for your app's package name to find its power consumption. This gives you real data to put on your poster!

Phase 3: User Experience Survey

7

Design a Simple User Survey

Create a Google Form with 8-10 questions. Here are suggested questions:

  1. Do you currently use dark mode on your phone? (Yes/No)
  2. Why do you prefer dark/light mode? (Multiple choice)
  3. Show both screenshots: Which version is easier to read? (Light/Dark/Same)
  4. Rate the readability of the dark mode version (1-5 scale)
  5. Rate the readability of the light mode version (1-5 scale)
  6. Do you experience eye strain with dark mode? (Never/Sometimes/Often)
  7. In what lighting conditions do you prefer dark mode? (Dark room/Any/Bright light)
  8. Do you believe dark mode saves battery? (Yes/No/Unsure)
üìä
Survey tip: Aim for at least 15-20 responses. Share it with classmates, friends, or family. Even a small sample makes your poster more convincing than no data at all!

üß™ Possible Experiments

Pick 2-3 experiments that interest you the most. Each one gives you data for your poster.

A

Battery Drain Comparison

Question: How much battery does dark mode actually save?

Method: Run your app in light and dark mode for equal time periods at different brightness levels (30%, 50%, 100%). Record battery percentage before and after each session.

Expected: At 100% brightness on OLED, dark mode may save 30-40%. At 30%, the saving drops to 3-9%.

B

User Preference Survey

Question: What percentage of users prefer dark mode, and why?

Method: Distribute your Google Form survey to 15-20+ people. Analyze the results using Google Sheets charts.

Poster visual: Pie charts showing preference percentages and bar charts showing reasons for preference.

C

True Black vs. Dark Gray Comparison

Question: Is true black (#000000) better than dark gray (#121212)?

Method: Create two dark mode variants ‚Äî one with true black, one with dark gray. Show both to users and ask which they find more comfortable. Test readability with a timed reading task.

Why interesting: Most users assume "blacker = better" but research suggests dark gray is actually more comfortable!

D

Readability Under Different Lighting

Question: Is dark mode actually better in low light?

Method: Have users read the same content in light mode vs. dark mode in (a) a bright room and (b) a dark room. Time how long it takes them to find specific information. Ask them to rate comfort.

Expected: Dark mode tends to be preferred in dark rooms; light mode is often more readable in bright environments.

E

Contrast Ratio Analysis

Question: Do popular apps meet accessibility contrast standards in dark mode?

Method: Use an online contrast checker (search: "WebAIM contrast checker"). Input the dark mode background and text colors from 5 popular apps. Check if they meet WCAG AA (4.5:1) standards.

Poster visual: A table showing pass/fail for each app ‚Äî very compelling data!

üîÑ Alternative Approaches

Option 1: Pure Literature Review + Survey

If battery testing is difficult (no physical OLED device), focus on:

  • Summarizing findings from the Purdue study and other published research
  • Running a user preference survey with your classmates
  • Showing implementation code examples (even if you don't measure battery yourself)

Option 2: App Audit Approach

Instead of building your own app, audit existing apps:

  • Pick 5-8 popular apps (YouTube, Twitter/X, Instagram, WhatsApp, etc.)
  • Screenshot their dark mode implementations
  • Analyze: Do they use true black or dark gray? What colors do they use for text? Do they meet contrast guidelines?
  • Rate each app's dark mode quality and explain why

Option 3: Accessibility Focus

Combine dark mode with accessibility research:

  • Research how dark mode affects users with visual impairments
  • Test with Android's TalkBack screen reader ‚Äî does dark mode change the experience?
  • Explore the "High Contrast" accessibility setting and how it interacts with dark mode

‚ö†Ô∏è Common Pitfalls & FAQ

"I don't have an OLED device to test battery"

That's okay! Focus on the Purdue study's published data and clearly state in your methodology: "Battery data is based on published research by [source] since physical OLED testing was not available." Then strengthen your poster with a user preference survey.

"My survey only got 10 responses"

Any data is better than no data! Just be transparent: "We surveyed 10 participants." Don't draw sweeping conclusions from small samples. Say "Our small sample suggests..." rather than "Users prefer..."

"Dark mode doesn't look different on the emulator"

The emulator uses your computer's LCD screen, so you won't see battery differences. But the visual theme should still change! Make sure you have both res/values/themes.xml and res/values-night/themes.xml and that your layouts use theme attributes like ?attr/colorSurface.

"Should I conclude that dark mode is always better?"

No! The best research acknowledges nuance. Dark mode has trade-offs: battery savings depend on screen type and brightness, readability can be worse in bright environments, and not all users prefer it. A nuanced conclusion shows mature thinking.
